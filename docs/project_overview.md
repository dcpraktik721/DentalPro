# Project Overview

## Зачем нужен этот repo

Этот репозиторий нужен как чистый operational workspace для всей дальнейшей работы по DentalPRO. Он отделяет текущую инженерную работу от legacy-каталогов и дает одно место для scripts, artifacts, registries, normalized data и Excel outputs.

## Какие слои DentalPRO уже исследуются

- browser-first runtime screens
- patient-related routes and tabs
- day schedule / visits / tickets
- report runtime / export / parity families
- financial and cashbox document layers
- proof-aware normalization and workbook building

## Какие типы задач repo будет закрывать

- live probes и forensic passes
- runtime/API comparison
- normalized dataset building
- Excel packs и management outputs
- report family validation
- knowledge consolidation и productization preparation
